# Winning strategy

## Primary track

Submit to **New Slack Agent** first. It has the cleanest fit and avoids Marketplace submission friction.

## Secondary story

Frame it as organization-ready, but do not rely on Marketplace approval unless you can complete it. Say:

"CampusOps is designed to become a Slack Agent for Organizations because every campus or small operations team can customize its routing and policy base."

## What judges should remember

CampusOps is not a generic support bot. It is an operations handoff agent for the people who get interrupted all day.

## MVP must-have

- Working Slack mention/DM or slash command
- Triage result with category and priority
- Safety-aware checklist
- Ticket created
- Handoff summary
- Architecture diagram
- 3-minute demo video

## Stretch features

1. Real-Time Search API for "what did we say about X?"
2. Buttons: Mark resolved / escalate / assign owner
3. App Home dashboard with open queue
4. MCP connector to a policy/wiki source
5. Marketplace submission

## Best demo prompts

```text
@CampusOps student cannot log in after password reset and has class in 20 minutes
@CampusOps printer says access denied in the library
@CampusOps resident says room heater is not working and room is freezing
@CampusOps what did we say last week about printer access?
@CampusOps handoff
```

## Pitch line

"CampusOps turns messy campus requests into safe next actions and clean shift handoffs, directly inside Slack."
